01نظرة عامةOverview

إطار RAG متكامل من البداية إلى النهاية، مبنيّ فعلاً من الصفر — وليس غلافاً حول LangChain. المسار الكامل: تحليل المستندات (PDF، DOCX، TXT، HTML، EPUB، PPTX، XLSX، والصور عبر OCR، والصوت عبر STT) ← تقطيع (8+ استراتيجيات) ← embeddings ← مخزن متّجهات ← بحث هجين (متّجهات كثيفة + BM25 يُدمَجان بالـ Reciprocal-Rank Fusion) ← إعادة ترتيب بـ cross-encoder ← توليد RAG باستشهادات على مستوى المقطع. An end-to-end RAG framework, genuinely built from scratch — not a LangChain wrapper. The pipeline: document parsing (PDF, DOCX, TXT, HTML, EPUB, PPTX, XLSX, images via OCR, audio via STT) → chunking (8+ strategies) → embeddings → vector store → hybrid search (dense vectors + BM25 fused with Reciprocal-Rank Fusion) → cross-encoder reranking → RAG generation with chunk-level citations.

يشحن الإطار واجهة سطر أوامر tadweer (init/add/index/search/status/eval)، ومعالج واجهة سطح مكتب بـ PyQt6، وخادم REST بـ FastAPI، وأهداف تصدير ONNX + حزمة مدمجة، ومنظومة تقييم (MRR، Recall@k، nDCG@k). دعم متعدّد اللغات مدمج في كامل المسار وليس مُلصقاً عليه لاحقاً. الحالة: الإصدار v0.2.0، المرحلة الثالثة مكتملة في معظمها. It ships a tadweer CLI (init/add/index/search/status/eval), a PyQt6 desktop GUI wizard, a FastAPI REST server, ONNX + embedded-bundle export targets, and an evaluation harness (MRR, Recall@k, nDCG@k). Multilingual support is baked in across the whole pipeline, not bolted on. Status: v0.2.0, Phase 3 mostly complete.

03المعماريةArchitecture

معمارية نظيفة متعدّدة الطبقات + انعكاس الاعتماديات (DIP) — النواة تستورد صفر اعتماديات خارجيةLayered clean architecture + dependency inversion — the core imports zero external deps

// clean / layered architecture · dependency inversion pivot

┌─ Presentation      tadweer CLI (Typer+Rich) · PyQt6 desktop wizard
│
├─ Application       pipeline orchestration · config · wiring (DI)
│
├─ Domain           models · interfaces (ABCs) · pure logic
│   └─ imports ZERO external deps  ← the dependency-inversion pivot
│
└─ Infrastructure   parsers · model loading · DB adapters
    heavy libs (torch · chromadb · qdrant · openai)
    lazy-loaded via Poetry extras · live ONLY here, behind ABCs

Data pipeline:
  docs → parse → chunk → embed → store
       → search(dense + BM25 → RRF) → rerank → generate(+citations)
  • محور انعكاس الاعتماديات: طبقة المجال (Domain) تحوي النماذج والواجهات والمنطق الصافي وتستورد صفر اعتماديات خارجية؛ كل التفاصيل الثقيلة تنزل إلى البنية التحتية خلف واجهات مجرّدة (ABCs).Dependency-inversion pivot: the Domain layer holds models, interfaces, and pure logic and imports zero external deps; all heavy details drop down into Infrastructure behind ABCs.
  • تحميل كسول للمكتبات الثقيلة: torch وchromadb وqdrant وopenai تُحمَّل عند الحاجة فقط عبر Poetry extras، وتعيش حصراً في طبقة البنية التحتية.Lazy-loaded heavy libs: torch, chromadb, qdrant, and openai are loaded only when needed via Poetry extras, and live exclusively in the Infrastructure layer.
  • مسار بيانات واضح: مستندات ← تحليل ← تقطيع ← embed ← تخزين ← بحث (كثيف + BM25 ← RRF) ← إعادة ترتيب ← توليد مع استشهادات.Explicit data pipeline: docs → parse → chunk → embed → store → search (dense + BM25 → RRF) → rerank → generate with citations.

04أبرز الميزات التقنيةKey technical features

01

تصميم متعدّد اللغاتLanguage-agnostic design

تطبيع الحركات والألف والياء والتطويل (قابل للضبط لكل وضع بحث)، كشف حدود الجمل الواعي باللغة، وكشف الترميز للنصوص المختلطة عربي/إنجليزي — ولكل مرحلة نصّية حالات اختبار خاصة بها.Diacritic/alef/ya/tatweel normalization (configurable per search mode), language-aware sentence boundary detection, and encoding detection for mixed Arabic/English scripts — every text stage has dedicated test cases.

02

بحث هجين بـ Reciprocal-Rank FusionHybrid search with Reciprocal-Rank Fusion

يدمج المتّجهات الكثيفة + BM25 مخصّص؛ على مجموعة الاختبار: متّجه MiniLM 0.940 MRR ← هجين 0.986 ← هجين + إعادة ترتيب 1.000.Combines dense vectors + a custom BM25; on the test corpus: MiniLM vector 0.940 MRR → hybrid 0.986 → hybrid+rerank 1.000.

03

إعادة ترتيب على مرحلتينTwo-stage reranking

استرجاع المرحلة الأولى ثم cross-encoder (BAAI/bge-reranker-v2-m3) لرفع الدقّة؛ ونتائج المُعيد ترتيبها مُخزَّنة مؤقتاً لكل (استعلام، مقطع).First-stage retrieval, then a cross-encoder (BAAI/bge-reranker-v2-m3) for precision; reranker results are cached per (query, chunk).

04

+8 استراتيجيات تقطيع8+ chunking strategies

تكراري، جُملي، رمزي (token)، واعٍ بالمستند، دلالي (كشف تحوّل الموضوع عبر cosine)، هرمي، متعدّد المتّجهات، وsmall-to-big (يحلّ مشكلة فقدان السياق).Recursive, sentence, token, document-aware, semantic (topic-shift detection via cosine), hierarchical, multi-vector, and small-to-big (solves lost-context).

05

فهرسة آمنة ضد الأعطال بنقاط تحقّقCrash-safe indexing with manifest checkpointing

تُكتَب نتيجة كل ملف (البصمة، المقاطع، الحالة) تدريجياً؛ وإعادة التشغيل تستأنف من حيث توقّفت وتتخطّى الملفات غير المتغيّرة.Each file's result (fingerprint, chunks, status) is written incrementally; re-runs resume where they stopped and skip unchanged files.

06

أهداف تصدير مكتفية ذاتياًSelf-contained export targets

نماذج ONNX (بلا torch وقت الاستدلال)، حِزَم مدمجة (vectors.f32 + chunks.json + meta، تُقرأ ببايثون صرف)، وخادم REST بـ FastAPI مع حقن اعتماديات.ONNX models (no torch at inference), embedded bundles (vectors.f32 + chunks.json + meta, readable with pure Python), and a FastAPI REST server with dependency injection.

05قرارات هندسيةEngineering decisions

امتلاك الطبقات التي تهمّOwning the layers that matter
نواة المجال تستورد صفر اعتماديات خارجية؛ مخازن المتّجهات (Chroma/Qdrant/FAISS) والـ embedders (sentence-transformers) ونماذج اللغة (Ollama/OpenAI/Anthropic/Gemini) تجلس جميعها خلف واجهات (ABCs)، وتُحمَّل كسولاً عبر Poetry extras. هذا يجعل النظام قابلاً للاختبار بـ fakes وقابلاً للتوسعة من أطراف ثالثة. ويُفرَض ذلك بفحص أنواع mypy الصارم. The domain core imports zero external dependencies; vector stores (Chroma/Qdrant/FAISS), embedders (sentence-transformers), and LLMs (Ollama/OpenAI/Anthropic/Gemini) all sit behind interfaces (ABCs), lazy-loaded via Poetry extras. This makes the system testable with fakes and pluggable for third parties — enforced with mypy strict typing.
الحالة بصراحةHonest status
الإصدار v0.2.0 هو beta وليس v1.0. واجهة PyQt6 مُتحقَّق منها بلا شاشة (Qt offscreen) لكنها لم تُختبر بعد على شاشة حقيقية؛ ومجموعة الـ benchmark المنشورة صغيرة (12 مستنداً، 36 سؤالاً — توضيحية، شغّل منظومة التقييم على بياناتك أنت)؛ وخطوة النشر على PyPI لا تزال يدوية. الشفافية أولاً. v0.2.0 is beta, not v1.0. The PyQt6 GUI is validated headless (offscreen Qt) but not yet exercised on a real display; the published benchmark corpus is small (12 docs, 36 questions — illustrative, run the eval harness on your own data); the PyPI publish step is still manual. Be transparent.
