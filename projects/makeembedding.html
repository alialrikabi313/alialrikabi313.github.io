01نظرة عامةOverview

إطار RAG متكامل من البداية إلى النهاية، مبنيّ فعلاً من الصفر لا غلافاً حول LangChain. والمسار كامل: تحليل المستندات (PDF، DOCX، TXT، HTML، EPUB، PPTX، XLSX، والصور عبر OCR، والصوت عبر STT) ← تقطيع (أكثر من 8 استراتيجيات) ← embeddings ← مخزن متّجهات ← بحث هجين (متّجهات كثيفة + BM25 يُدمَجان عبر Reciprocal-Rank Fusion) ← إعادة ترتيب عبر cross-encoder ← توليد RAG باستشهادات على مستوى المقطع. An end-to-end RAG framework, genuinely built from scratch — not a LangChain wrapper. The pipeline: document parsing (PDF, DOCX, TXT, HTML, EPUB, PPTX, XLSX, images via OCR, audio via STT) → chunking (8+ strategies) → embeddings → vector store → hybrid search (dense vectors + BM25 fused with Reciprocal-Rank Fusion) → cross-encoder reranking → RAG generation with chunk-level citations.

يأتي الإطار بواجهة سطر أوامر tadweer (init/add/index/search/status/eval)، ومعالج سطح مكتب بـ PyQt6، وخادم REST بـ FastAPI، وصيغ تصدير ONNX وحزمة مدمجة، ومنظومة تقييم (MRR، Recall@k، nDCG@k). ودعم اللغات المتعدّدة مدمج في كامل المسار لا مُلصق عليه لاحقاً. أما الحالة فهي: الإصدار v0.2.0، والمرحلة الثالثة شبه مكتملة. It ships a tadweer CLI (init/add/index/search/status/eval), a PyQt6 desktop GUI wizard, a FastAPI REST server, ONNX + embedded-bundle export targets, and an evaluation harness (MRR, Recall@k, nDCG@k). Multilingual support is baked in across the whole pipeline, not bolted on. Status: v0.2.0, Phase 3 mostly complete.

معمارية نظيفة متعدّدة الطبقات مع انعكاس الاعتماديات (DIP)، والنواة لا تستورد أيّ اعتمادية خارجيةLayered clean architecture + dependency inversion — the core imports zero external deps

// clean / layered architecture · dependency inversion pivot

┌─ Presentation      tadweer CLI (Typer+Rich) · PyQt6 desktop wizard
│
├─ Application       pipeline orchestration · config · wiring (DI)
│
├─ Domain           models · interfaces (ABCs) · pure logic
│   └─ imports ZERO external deps  ← the dependency-inversion pivot
│
└─ Infrastructure   parsers · model loading · DB adapters
    heavy libs (torch · chromadb · qdrant · openai)

  docs → parse → chunk → embed → store
       → search(dense + BM25 → RRF) → rerank → generate(+citations)
  • محور انعكاس الاعتماديات: طبقة domain تضمّ النماذج والواجهات والمنطق الصافي ولا تستورد أيّ اعتمادية خارجية، فيما تهبط كل التفاصيل الثقيلة إلى البنية التحتية خلف واجهات مجرّدة (ABCs).Dependency-inversion pivot: the Domain layer holds models, interfaces, and pure logic and imports zero external deps; all heavy details drop down into Infrastructure behind ABCs.
  • تحميل كسول للمكتبات الثقيلة: لا تُحمَّل torch وchromadb وqdrant وopenai إلا عند الحاجة عبر Poetry extras، وتبقى محصورة في طبقة البنية التحتية.Lazy-loaded heavy libs: torch, chromadb, qdrant, and openai are loaded only when needed via Poetry extras, and live exclusively in the Infrastructure layer.
  • مسار بيانات واضح: مستندات ← تحليل ← تقطيع ← embed ← تخزين ← بحث (كثيف + BM25 ← RRF) ← إعادة ترتيب ← توليد مع استشهادات.Explicit data pipeline: docs → parse → chunk → embed → store → search (dense + BM25 → RRF) → rerank → generate with citations.

04أبرز الميزات التقنيةKey technical features

01

تصميم متعدّد اللغاتLanguage-agnostic design

تطبيع الحركات والألف والياء والتطويل (قابل للضبط في كل وضع بحث)، وكشف حدود الجمل بوعي باللغة، وكشف الترميز للنصوص المختلطة عربي/إنجليزي، ولكل مرحلة نصّية حالات اختبار خاصة بها.Diacritic/alef/ya/tatweel normalization (configurable per search mode), language-aware sentence boundary detection, and encoding detection for mixed Arabic/English scripts — every text stage has dedicated test cases.

02

بحث هجين بـ Reciprocal-Rank FusionHybrid search with Reciprocal-Rank Fusion

يدمج المتّجهات الكثيفة مع BM25 مخصّص؛ وعلى مجموعة تقييم توضيحية صغيرة (12 مستنداً، 36 سؤالاً) يرتفع الـ MRR من 0.940 (متّجه MiniLM) إلى 0.986 (هجين) ثم 1.000 (هجين + إعادة ترتيب) — أرقام دالّة على الاتجاه لا قياس على مقياس مرجعي كبير.Combines dense vectors + a custom BM25; on a small illustrative eval set (12 docs, 36 questions), MRR rises from 0.940 (MiniLM vector) → 0.986 (hybrid) → 1.000 (hybrid + rerank) — directional figures, not a large-benchmark result.

03

إعادة ترتيب على مرحلتينTwo-stage reranking

استرجاع في المرحلة الأولى، يليه cross-encoder (BAAI/bge-reranker-v2-m3) لرفع الدقّة؛ وتُخزَّن نتائج إعادة الترتيب مؤقتاً لكل (استعلام، مقطع).First-stage retrieval, then a cross-encoder (BAAI/bge-reranker-v2-m3) for precision; reranker results are cached per (query, chunk).

04

+8 استراتيجيات تقطيع8+ chunking strategies

تكراري، وجُملي، ورمزي (token)، وواعٍ بالمستند، ودلالي (يكشف تحوّل الموضوع عبر cosine)، وهرمي، ومتعدّد المتّجهات، وsmall-to-big (يحلّ مشكلة فقدان السياق).Recursive, sentence, token, document-aware, semantic (topic-shift detection via cosine), hierarchical, multi-vector, and small-to-big (solves lost-context).

05

فهرسة آمنة ضد الأعطال بنقاط تحقّقCrash-safe indexing with manifest checkpointing

تُكتَب نتيجة كل ملف (البصمة، والمقاطع، والحالة) تدريجياً؛ فتستأنف عمليةُ التشغيل المعادة من حيث توقّفت وتتخطّى الملفات التي لم تتغيّر.Each file's result (fingerprint, chunks, status) is written incrementally; re-runs resume where they stopped and skip unchanged files.

06

صيغ تصدير مكتفية ذاتياًSelf-contained export targets

نماذج ONNX (بلا torch وقت الاستدلال)، وحِزَم مدمجة (vectors.f32 + chunks.json + meta، تُقرأ ببايثون صرف)، وخادم REST بـ FastAPI مع حقن اعتماديات.ONNX models (no torch at inference), embedded bundles (vectors.f32 + chunks.json + meta, readable with pure Python), and a FastAPI REST server with dependency injection.

05قرارات هندسيةEngineering decisions

امتلاك الطبقات التي تهمّOwning the layers that matter
نواة domain لا تستورد أيّ اعتمادية خارجية؛ أما مخازن المتّجهات (Chroma/Qdrant/FAISS) والـ embedders (sentence-transformers) ونماذج اللغة (Ollama/OpenAI/Anthropic/Gemini) فتقع جميعها خلف واجهات (ABCs)، وتُحمَّل كسولاً عبر Poetry extras. وهذا يجعل النظام قابلاً للاختبار بـ fakes وقابلاً لأن توسّعه أطراف ثالثة، ويُفرَض ذلك بفحص أنواع mypy الصارم. The domain core imports zero external dependencies; vector stores (Chroma/Qdrant/FAISS), embedders (sentence-transformers), and LLMs (Ollama/OpenAI/Anthropic/Gemini) all sit behind interfaces (ABCs), lazy-loaded via Poetry extras. This makes the system testable with fakes and pluggable for third parties — enforced with mypy strict typing.
الحالة بصراحةHonest status
الإصدار v0.2.0 هو beta لا v1.0. وواجهة PyQt6 جرى التحقّق منها بلا شاشة (Qt offscreen) لكنها لم تُختبر بعد على شاشة حقيقية؛ ومجموعة الـ benchmark المنشورة صغيرة (12 مستنداً و36 سؤالاً، وهي توضيحية، فشغّل منظومة التقييم على بياناتك أنت)؛ وخطوة النشر على PyPI لا تزال يدوية. الشفافية أولاً. v0.2.0 is beta, not v1.0. The PyQt6 GUI is validated headless (offscreen Qt) but not yet exercised on a real display; the published benchmark corpus is small (12 docs, 36 questions — illustrative, run the eval harness on your own data); the PyPI publish step is still manual. Be transparent.
